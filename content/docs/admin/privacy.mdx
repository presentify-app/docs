---
title: Privacy Settings
description: Configure workspace visibility levels, booking privacy, feature toggles, and data retention.
---

Privacy settings control how much information is shared between workspace members and how long data is retained. Access them via **Settings** > **Privacy**.

## Colleague visibility

Choose how much presence and booking information is visible to colleagues:

| Level | Description |
|-------|-------------|
| **Full** | Everyone sees everyone's name, status, desk location, and booking info |
| **Restricted** | Team members and favorites see full details. Others see limited information. |
| **Minimal** | Only anonymous aggregates are shown. No individual presence or booking data visible. |

Individual members can further restrict their own visibility in their [Profile settings](/docs/employee/profile) (opt out of Who's In, hide desk on map, etc.).

## Booking privacy

| Setting | Default | Description |
|---------|---------|-------------|
| **Show Future Bookings** | On | Allow colleagues to see each other's upcoming bookings |
| **Allow Private Bookings** | On | Let members mark individual bookings as private |
| **Default Bookings to Private** | Off | New bookings are marked private by default |

Members can override the default in their profile settings and per-booking in the booking dialog.

## Feature toggles

Enable or disable workspace-wide features:

| Feature | Default | Description |
|---------|---------|-------------|
| **Guest Booking** | On | Allow booking desks for external guests |
| **Parking** | On | Enable the parking feature |
| **Who's In** | On | Enable the Who's In page |

Disabled features are hidden from the sidebar and become inaccessible to all members.

## Data retention (GDPR)

Configure how long historical data is kept:

- **Data Retention Period** — Default: 12 months (range: 1–120 months)
- After this period, presence entries and booking records are automatically anonymized
- Anonymization is irreversible — the data is stripped of personal identifiers

This supports GDPR Article 17 (Right to Erasure). Members can also request deletion of their own data through their profile settings.

### Product analytics data retention

In addition to the workspace-level retention period above, Presentify applies the following **global** retention policies to product analytics data automatically:

| Data type | Retention | Action |
|-----------|-----------|--------|
| **Analytics events** (visitor ID, IP address, user agent) | 24 months | Personal identifiers are anonymized — event records are kept for aggregate reporting |
| **Visitor attributions** (unlinked to a user account) | 6 months | Deleted entirely if the visitor record was never linked to a registered user |

These policies run system-wide and are not configurable per workspace.

## In-app satisfaction survey

Presentify may display an optional **satisfaction survey** (NPS) inside the app. The survey:

- Appears **at most once every 90 days** per user
- Can be **dismissed** at any time without submitting
- Stores a cooldown timestamp in the browser's local storage (`pfy_nps_last_shown`) — no cookie is set
- Submitted responses (score and optional free-text feedback) are sent to the analytics endpoint and subject to the retention policies described above

## Member-level overrides

Each member has personal privacy toggles that override workspace settings for their own data:

- **Show in Who's In** — Appear in the Who's In view
- **Show Desk on Map** — Show desk location on floor plans
- **Show in Colleagues Week** — Appear in the team calendar
- **Default Booking Private** — New bookings private by default

These are managed by the member in their [Profile](/docs/employee/profile), or by an admin in **Settings** > **Members** > member actions > **Edit Settings**.
