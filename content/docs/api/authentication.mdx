---
title: Authentication
description: How to authenticate with the Presentify REST API using API keys.
---

All API requests must include a valid API key in the `X-API-KEY` header.

## Request header

```
X-API-KEY: pk_your_api_key_here
```

## Creating an API key

See [API Keys](/docs/integrations/api-keys) for detailed instructions. In short:

1. Go to **Settings** > **API Keys**
2. Click **Create API Key**
3. Copy the key (shown once only)

## Workspace scoping

Each API key is bound to a specific workspace. All queries automatically filter data to that workspace — there is no way to access data from other workspaces.

## Run-as member

Each API key operates in the context of a specific member (the "run-as" member). This member must be:

- **Active** in the workspace (not invited or inactive)
- Part of the workspace associated with the key

The run-as member determines the context for write operations (e.g., who created a presence entry via the API).

## Requirements

- **Enterprise plan** — The API is gated behind the `api_access` feature
- **Valid API key** — Must exist in the database and not be revoked
- **Active run-as member** — The member associated with the key must be active

If any of these conditions are not met, the API returns a `401 UNAUTHORIZED` or `403 FORBIDDEN` error.

## Security best practices

- Store API keys in environment variables, never hardcode them
- Use HTTPS for all API requests (HTTP is not supported)
- Rotate keys periodically
- Revoke keys immediately if compromised
- Use a dedicated service account as the run-as member
