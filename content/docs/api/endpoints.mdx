All endpoints use the tRPC HTTP protocol. GET requests pass input as a JSON-encoded `input` query parameter. POST requests send input in the request body.

## Members

### `member.list` — List members

```bash
GET /api/v1/member.list?input={"status":"active"}
```

**Input (optional):**

| Parameter | Type | Description |
|-----------|------|-------------|
| `email` | string | Filter by exact email |
| `customId` | string | Filter by custom external ID |
| `status` | string | `"active"`, `"inactive"`, or `"invited"` |

**Response:** Array of member objects with `id`, `name`, `firstName`, `lastName`, `email`, `customId`, `role`, `status`, `language`, `timezone`, `departmentId`, `teamId`, `createdAt`.

### `member.getById` — Get member by ID

```bash
GET /api/v1/member.getById?input={"id":"member-uuid"}
```

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | Member UUID |

**Response:** Single member object with full details including `employmentStartDate`, `employmentEndDate`, `defaultLocationId`. Returns `null` if not found.

## Presence

### `presence.list` — List presence entries

```bash
GET /api/v1/presence.list?input={"startDate":"2026-03-01","endDate":"2026-03-31"}
```

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `startDate` | string | Yes | Start date (YYYY-MM-DD) |
| `endDate` | string | Yes | End date (YYYY-MM-DD) |
| `memberIds` | string[] | No | Filter by specific member UUIDs |

**Response:** Array of presence entries with `id`, `date`, `status`, `startAt`, `endAt`, `isHalfDay`, `halfDayPart`, `memberId`, `locationId`, `createdAt`.

**Status values:** `in_office`, `remote`, `on_the_road`, `customer_site`, `training_event`, `onsite_without_desk`, `half_day_office`, `half_day_remote`, `away`, `unknown`

### `presence.upsert` — Create or update presence

```bash
POST /api/v1/presence.upsert
Content-Type: application/json

{
  "memberId": "member-uuid",
  "date": "2026-03-25",
  "status": "in_office",
  "locationId": "location-uuid"
}
```

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `memberId` | string | Yes | Member UUID |
| `date` | string | Yes | Date (YYYY-MM-DD) |
| `status` | string | Yes | Presence status |
| `locationId` | string | No | Office location UUID |
| `isHalfDay` | boolean | No | Half-day flag (default: false) |
| `halfDayPart` | string | No | `"morning"` or `"afternoon"` |

**Response:** `{ "id": "entry-uuid" }` — The created or updated entry ID.

If an entry already exists for the same member and date, it is updated. Otherwise, a new entry is created.

### `presence.delete` — Delete presence entry

```bash
POST /api/v1/presence.delete
Content-Type: application/json

{ "id": "entry-uuid" }
```

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | Presence entry UUID |

**Response:** `{ "success": true }`

## Bookings

### `booking.list` — List bookings

```bash
GET /api/v1/booking.list?input={"startDate":"2026-03-01","endDate":"2026-03-31","resourceType":"desk"}
```

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `startDate` | string | Yes | Start date (YYYY-MM-DD) |
| `endDate` | string | Yes | End date (YYYY-MM-DD) |
| `memberId` | string | No | Filter by member UUID |
| `resourceType` | string | No | `"desk"` or `"parking_spot"` |
| `resourceId` | string | No | Filter by specific desk/spot UUID |
| `status` | string | No | `"confirmed"`, `"checked_in"`, `"completed"`, `"cancelled"`, `"no_show"`, `"auto_released"` |

**Response:** Array of booking objects with `id`, `resourceType`, `resourceId`, `memberId`, `date`, `startAt`, `endAt`, `isFullDay`, `status`, `checkedInAt`, `checkedOutAt`, `checkInMethod`, `bookedByMemberId`, `bookedForMemberId`, `isGuestBooking`, `guestName`, `guestEmail`, `guestCompany`, `isPrivate`, `notes`, `createdAt`.

### `booking.getById` — Get booking by ID

```bash
GET /api/v1/booking.getById?input={"id":"booking-uuid"}
```

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | Booking UUID |

**Response:** Single booking object with full details including `cancelledAt`, `cancelReason`. Returns `null` if not found.

### `booking.cancel` — Cancel a booking

```bash
POST /api/v1/booking.cancel
Content-Type: application/json

{
  "id": "booking-uuid",
  "reason": "No longer needed"
}
```

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | Booking UUID |
| `reason` | string | No | Cancellation reason (default: "Cancelled via API") |

**Response:** `{ "success": true }`

Only bookings with status `confirmed`, `checked_in`, or `completed` can be cancelled.
