title: Fehler

---

Die API nutzt Standard-HTTP-Statuscodes und tRPC-Fehlercodes, um das Ergebnis einer Anfrage anzuzeigen.

## Fehler-Antwortformat

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "The requested resource was not found."
  }
}
```

## HTTP-Statuscodes

| Status | Code | Beschreibung |
|--------|------|-------------|
| **200** | OK | Anfrage erfolgreich |
| **400** | `BAD_REQUEST` | Ungültige Eingabe — prüfe deine Anfrageparameter |
| **401** | `UNAUTHORIZED` | Fehlender, ungültiger oder abgelaufener API-Schlüssel |
| **403** | `FORBIDDEN` | API-Schlüssel gültig, aber Zugriff verweigert (falscher Plan, inaktives Mitglied etc.) |
| **404** | `NOT_FOUND` | Ressource nicht gefunden |
| **409** | `CONFLICT` | Konflikt — z.B. doppelter Eintrag |
| **429** | `TOO_MANY_REQUESTS` | Rate Limit überschritten |
| **500** | `INTERNAL_SERVER_ERROR` | Serverfehler — kontaktiere Support bei anhaltendem Problem |

## Häufige Fehlerszenarien

### Authentifizierungsfehler (401)

- API-Schlüssel nicht im `X-API-KEY`-Header angegeben
- API-Schlüssel existiert nicht (widerrufen oder nie erstellt)
- Ausführen-als-Mitglied ist inaktiv oder nicht gefunden

### Autorisierungsfehler (403)

- Workspace hat keinen Enterprise-Plan (`api_access`-Feature erforderlich)
- Versuch, auf Daten außerhalb des Workspace des API-Schlüssels zuzugreifen

### Validierungsfehler (400)

- Fehlende Pflichtfelder (`startDate`, `endDate`, `id` etc.)
- Ungültiges Datumsformat (erwartet `YYYY-MM-DD`)
- Ungültiger Enum-Wert (z.B. unbekannter Status)
- Mitglied-ID gehört nicht zum Workspace

### Nicht-gefunden-Fehler (404)

- Mitglied, Standort, Buchung oder Anwesenheitseintrag nicht gefunden
- ID ist gültig, aber die Ressource existiert nicht in diesem Workspace

### Rate-Limit-Fehler (429)

- Mehr als 150 Anfragen pro 60 Sekunden von deiner IP
- Warten und mit exponentiellem Backoff erneut versuchen
