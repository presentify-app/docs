Die Presentify REST API v1 bietet programmatischen Zugriff auf deine Workspace-Daten. Nutze sie, um eigene Integrationen zu bauen, Workflows zu automatisieren und Daten zu exportieren.

> Die REST API erfordert einen **Enterprise**-Plan und einen [API-Schlüssel](/docs/integrations/api-keys).

## Basis-URL

```
https://app.presentify.org/api/v1
```

## Authentifizierung

Füge deinen API-Schlüssel im `X-API-KEY`-Header bei jeder Anfrage ein:

```bash
curl -H "X-API-KEY: pk_dein_schluessel" \
  "https://app.presentify.org/api/v1/member.list?input={}"
```

Siehe [API-Schlüssel](/docs/integrations/api-keys) für die Einrichtung.

## Verfügbare Endpunkte

| Gruppe | Endpunkte | Beschreibung |
|--------|-----------|-------------|
| **Mitglieder** | `member.list`, `member.getById` | Workspace-Mitglieder auflisten und abrufen |
| **Standorte** | `location.list`, `location.getById` | Bürostandorte auflisten und abrufen |
| **Anwesenheit** | `presence.list`, `presence.upsert`, `presence.delete` | Anwesenheitseinträge verwalten |
| **Buchungen** | `booking.list`, `booking.getById`, `booking.cancel` | Schreibtisch- und Parkplatzbuchungen verwalten |

Die detaillierte Endpunkt-Referenz mit Parameter-Schemas, Antwortformaten und Beispielen findest du in der Sidebar.

## Rate Limits

- **150 Anfragen** pro 60 Sekunden pro IP-Adresse
- Gibt `429 Too Many Requests` zurück bei Überschreitung

## Webhooks

Für Echtzeit-Event-Benachrichtigungen siehe [Webhooks](/docs/integrations/webhooks) (Enterprise).

## Antwortformat

Alle Antworten nutzen das tRPC JSON-RPC Format:

```json
{
  "result": {
    "data": [ ... ]
  }
}
```

Daten verwenden ISO 8601 Format (`2026-03-20T14:30:00Z`). Nur-Datum-Felder nutzen `YYYY-MM-DD`.
