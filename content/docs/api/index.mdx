The Presentify REST API v1 provides programmatic access to your workspace data. Use it to build custom integrations, automate workflows, and export data.

> The REST API requires an **Enterprise** plan and an [API key](/docs/integrations/api-keys).

## Base URL

```
https://app.presentify.org/api/v1
```

## Authentication

Include your API key in the `X-API-KEY` header on every request:

```bash
curl -H "X-API-KEY: pk_your_key_here" \
  "https://app.presentify.org/api/v1/member.list?input={}"
```

See [API Keys](/docs/integrations/api-keys) for setup instructions.

## Available endpoints

| Group | Endpoints | Description |
|-------|-----------|-------------|
| **Members** | `member.list`, `member.getById` | List and get workspace members |
| **Locations** | `location.list`, `location.getById` | List and get office locations |
| **Presence** | `presence.list`, `presence.upsert`, `presence.delete` | Manage presence entries |
| **Bookings** | `booking.list`, `booking.getById`, `booking.cancel` | Manage desk and parking bookings |

Browse the endpoint reference in the sidebar for detailed parameter schemas, response formats, and examples.

## Rate limits

- **150 requests** per 60 seconds per IP address
- Returns `429 Too Many Requests` when exceeded

## Webhooks

For real-time event notifications, see [Webhooks](/docs/integrations/webhooks) (Enterprise).

## Response format

All responses use the tRPC JSON-RPC format:

```json
{
  "result": {
    "data": [ ... ]
  }
}
```

Dates use ISO 8601 format (`2026-03-20T14:30:00Z`). Date-only fields use `YYYY-MM-DD`.
