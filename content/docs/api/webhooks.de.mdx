## Payload-Struktur

```json
{
  "id": "delivery-uuid",
  "type": "desk_booking.created",
  "occurred_at": "2026-03-20T14:30:00.000Z",
  "data": { ... }
}
```

| Feld | Typ | Beschreibung |
|------|-----|-------------|
| `id` | string | Eindeutige Delivery-ID |
| `type` | string | Event-Typ |
| `occurred_at` | ISO 8601 | Wann das Event aufgetreten ist |
| `data` | object | Event-spezifischer Payload |

## Signaturverifikation

Jede Webhook-Anfrage enthält einen `X-Presentify-Signature`-Header mit einem HMAC-SHA256-Hex-Digest des rohen Request-Bodys, signiert mit deinem Webhook-Secret.

**Node.js:**

```javascript
const crypto = require("crypto");

function verifyWebhook(rawBody, signature, secret) {
  const expected = crypto
    .createHmac("sha256", secret)
    .update(rawBody, "utf8")
    .digest("hex");
  return crypto.timingSafeEqual(
    Buffer.from(expected),
    Buffer.from(signature)
  );
}
```

**Python:**

```python
import hmac, hashlib

def verify_webhook(raw_body: bytes, signature: str, secret: str) -> bool:
    expected = hmac.new(
        secret.encode(), raw_body, hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(expected, signature)
```

Verifiziere immer die Signatur, bevor du den Webhook-Payload verarbeitest.

## Retry-Zeitplan

Fehlgeschlagene Zustellungen (Nicht-2xx oder Timeout nach 10s) werden bis zu 5 Mal wiederholt:

1 Min → 5 Min → 30 Min → 2 Stunden → 12 Stunden

Siehe [Webhooks Integrations-Guide](/docs/integrations/webhooks) für vollständige Retry-Details.
