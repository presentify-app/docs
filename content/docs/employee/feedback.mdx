Presentify includes a built-in Feedback board where every team member can share ideas, vote on features, and discuss priorities directly inside the app.

## Accessing the Feedback board

- Click the **Feedback** icon (chat bubble with plus) in the top navigation bar to open a slide-over panel.
- Alternatively, navigate to **Feedback** in the sidebar to open the full-page view.

Both views show the same content — the panel is handy for quick browsing, while the full page gives you more room.

## Browsing ideas

Ideas are listed in two sort orders:

- **Top** — Most votes first
- **Newest** — Most recently submitted first

Switch between them using the tabs at the top.

Use the **search bar** to filter ideas by keyword. The search checks both title and description.

Each idea card shows:

- Title and a two-line description preview
- Vote count and comment count
- Assigned topics (colored category tags)
- A status badge (if the idea is not in the default *Submitted* state)

## Submitting a new idea

1. Click **New Idea** (available in both the panel and the full page)
2. Enter a **Title** (3–200 characters) and a **Description** (10–5,000 characters)
3. Optionally select up to **3 Topics** (colored category tags set by your admin)
4. As you type the title, a *Similar ideas* list may appear — check it to avoid duplicates before submitting
5. Click **Submit Idea**

Your idea is automatically upvoted by you upon creation.

## Voting

- Click the **upvote button** (chevron-up with a count) on any idea card or detail view to add your vote. Click it again to remove your vote.
- You can only cast one vote per idea.

## Comments

1. Open an idea to see its full description and comments
2. Type in the comment box at the bottom and click the **Send** button to post
3. You can **edit** or **delete** your own comments using the three-dot menu that appears on hover

## Status badges

| Badge | Meaning |
|---|---|
| **Submitted** | Default state — idea is awaiting review |
| **Under Review** | The team is actively evaluating this idea |
| **Planned** | The feature has been scheduled for development |
| **Shipped** | The feature has been released |

Statuses are set by workspace Owners. You'll see the current status on each idea card and in the detail view.
