Webhooks send HTTP POST requests to your server when events happen in Presentify. Use them to integrate with external systems like Slack, HR tools, or custom dashboards.

> Webhooks require an **Enterprise** plan.

## Creating a webhook endpoint

1. Go to **Settings** > **Webhooks**
2. Click **Add Endpoint**
3. Configure:
   - **URL** (required) — Your HTTPS endpoint
   - **Description** (optional) — Max 200 characters
   - **Events** (required) — Select which events to receive
4. Click **Save**

The webhook **secret** is shown once after creation. Copy it immediately — you'll need it for signature verification.

You can create up to **5 webhook endpoints** per workspace.

## Payload format

Every webhook delivery wraps the event data in a standard envelope:

```json
{
  "id": "delivery-uuid",
  "type": "desk_booking.created",
  "occurred_at": "2026-03-20T14:30:00.000Z",
  "data": {
    "booking_id": "...",
    "member_id": "...",
    "desk_id": "...",
    "date": "2026-03-21",
    "status": "confirmed"
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique delivery ID |
| `type` | `string` | Event type (e.g., `desk_booking.created`) |
| `occurred_at` | `string` | ISO 8601 timestamp of when the event occurred |
| `data` | `object` | Event-specific payload (see [Event type details](#event-type-details)) |

## HTTP headers

| Header | Description |
|--------|-------------|
| `X-Presentify-Signature` | HMAC-SHA256 hex digest of the payload |
| `X-Presentify-Event` | Event type (e.g., `desk_booking.created`) |
| `X-Presentify-Delivery` | Unique delivery ID |
| `X-Presentify-Retry` | Retry attempt number (only on retries) |
| `Content-Type` | `application/json` |

## Signature verification

Each webhook request is signed with your workspace's webhook secret using HMAC-SHA256. Verify the `X-Presentify-Signature` header to ensure the request is authentic.

**Node.js:**

```javascript
const crypto = require("crypto");

function verifySignature(secret, payload, signature) {
  const computed = crypto
    .createHmac("sha256", secret)
    .update(payload, "utf8")
    .digest("hex");
  return computed === signature;
}

// Usage in Express:
app.post("/webhook", (req, res) => {
  const sig = req.headers["x-presentify-signature"];
  if (!verifySignature(WEBHOOK_SECRET, req.rawBody, sig)) {
    return res.status(401).send("Invalid signature");
  }
  // Process event...
  res.status(200).send("OK");
});
```

**Python:**

```python
import hmac
import hashlib

def verify_signature(secret: str, payload: str, signature: str) -> bool:
    computed = hmac.new(
        secret.encode("utf-8"),
        payload.encode("utf-8"),
        hashlib.sha256,
    ).hexdigest()
    return hmac.compare_digest(computed, signature)

# Usage in Flask:
@app.route("/webhook", methods=["POST"])
def webhook():
    sig = request.headers.get("X-Presentify-Signature")
    if not verify_signature(WEBHOOK_SECRET, request.data.decode(), sig):
        return "Invalid signature", 401
    # Process event...
    return "OK", 200
```

## Retry behavior

If your endpoint returns a non-2xx status code or times out (10 seconds), Presentify retries the delivery:

| Attempt | Delay |
|---------|-------|
| 1st retry | 1 minute |
| 2nd retry | 5 minutes |
| 3rd retry | 30 minutes |
| 4th retry | 2 hours |
| 5th retry | 12 hours |

After 5 failed retries, the delivery is marked as failed. Each attempt is logged in the delivery history.

## Test delivery

Click **Test** on a webhook endpoint to send a test payload. This verifies your endpoint is reachable and responding correctly.

## Delivery log

Each endpoint shows recent deliveries with:

- Event type and delivery ID
- Response status code
- Success/failure status
- Attempt count
- Timestamp

Use the delivery log to debug integration issues.

## Best practices

- **Respond quickly** — Return `200 OK` immediately, then process the event asynchronously.
- **Handle idempotency** — Use the `X-Presentify-Delivery` header to deduplicate events. You may receive the same event more than once.
- **Always verify signatures** — Verify the signature before processing the payload to ensure the request is authentic.
